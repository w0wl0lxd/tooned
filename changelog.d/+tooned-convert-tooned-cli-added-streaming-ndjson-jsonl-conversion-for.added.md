**tooned-convert / tooned-cli:** added streaming NDJSON/JSONL conversion for large inputs. `tooned convert --to tron` and the default adaptive path now stream NDJSON/JSONL inputs (when the format is explicit via `--format-hint ndjson` or the file extension is `.jsonl`/`.ndjson`). For `--to tron` forced, streaming writes to a temp file and then promotes it atomically for file output or copies to stdout for stdout output. For the default adaptive case, streaming writes to a temp file, compares output size vs input size using the margin check, and discards the temp and passes through the original input if not smaller enough. On parse/IO error, falls back to passthrough of the original input (for stdin, spools stdin to a temp file first so it can be retried/copied). Only uses streaming when the input is large (above `max_input_bytes`) or the user explicitly forced `--to tron` with an NDJSON hint/extension. Small NDJSON inputs continue to use the bounded path.
