Metrics aggregates (`summary`, `heatmap`, `per_surface`, `top_files`, `cost_summary`) now compute byte/token sums in floating point so a running total exceeding `i64::MAX` no longer makes the query fail with a SQLite integer-overflow error; `Store::open` re-checks for a symlink after opening (mirroring the index crate's `open_index` double-check) and `now_unix` saturates at `i64::MAX` instead of wrapping.
