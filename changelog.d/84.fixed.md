`tooned-cli` no longer emits `additionalContext` for Codex, Devin, or Droid `PostToolUse` hooks. `additionalContext` keeps the original JSON in context and appends the TOON, inflating total token count rather than reducing it. Codex now uses `continue: false` + `reason` feedback to replace the model-visible tool result with TOON; Devin and Droid pass through because their `PostToolUse` protocols do not support output replacement. Use `tooned wrap -- <cmd>` or `... | tooned pipe` with Devin/Droid (and for any command on other agents) when TOON-only output is required. README and evidence docs were updated to describe the per-protocol behavior and the re-performed mismatch test.
